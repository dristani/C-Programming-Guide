15. Unions

Understanding unions in C programming

Introduction to Unions

Unions in C are similar to structures but with a key difference: all members of a union share the same memory location. This means that only one member can contain a value at any given time. Unions are useful when you need to store different types of data in the same memory location.

Union Operations

Operation Description Example
Declaration Defining a union type union Data { ... };
Initialization Creating union variables union Data d1;
Access Accessing union members d1.i
Pointer Access Accessing via union pointer ptr->i
Array of Unions Creating multiple unions union Data arr[10];
Nested Unions Unions within structures struct { union { ... } };

Example Program

#include <stdio.h>
#include <string.h>

// Union definition
union Data {
    int i;
    float f;
    char str[20];
};

// Structure with nested union
struct Student {
    char name[50];
    union {
        int roll_no;
        char section;
    } id;
};

int main() {
    // Union variable declaration and initialization
    union Data data;
    
    // Using integer member
    data.i = 10;
    printf("data.i: %d\n", data.i);
    
    // Using float member (overwrites previous value)
    data.f = 220.5;
    printf("data.f: %.1f\n", data.f);
    
    // Using string member (overwrites previous value)
    strcpy(data.str, "C Programming");
    printf("data.str: %s\n", data.str);
    
    // Structure with nested union
    struct Student s1;
    strcpy(s1.name, "John");
    
    // Using roll number
    s1.id.roll_no = 101;
    printf("\nStudent Details:\n");
    printf("Name: %s\n", s1.name);
    printf("Roll No: %d\n", s1.id.roll_no);
    
    // Using section (overwrites roll number)
    s1.id.section = 'A';
    printf("Section: %c\n", s1.id.section);
    
    // Array of unions
    union Data arr[3];
    for(int i = 0; i < 3; i++) {
        arr[i].i = i * 10;
        printf("arr[%d].i: %d\n", i, arr[i].i);
    }
    
    // Union pointer
    union Data *ptr = &data;
    ptr->i = 100;
    printf("\nAccessing via pointer:\n");
    printf("ptr->i: %d\n", ptr->i);
    
    return 0;
}

Code Explanation

  • union Data { ... }; - Union definition with multiple member types
  • data.i = 10; - Storing integer value in union
  • data.f = 220.5; - Storing float value (overwrites previous value)
  • strcpy(data.str, "C Programming"); - Storing string (overwrites previous value)
  • struct Student { ... }; - Structure with nested union
  • s1.id.roll_no - Accessing union member within structure

Union Characteristics

  • All union members share the same memory location
  • Only one member can contain a value at a time
  • Size of union is equal to size of largest member
  • Unions can contain different data types
  • Unions can be nested within structures
  • Unions can be used for type conversion
  • Unions are useful for memory optimization

Union Best Practices

  • Use unions when you need to store different types in same memory
  • Keep track of which member is currently in use
  • Use unions for type conversion carefully
  • Consider memory alignment and padding
  • Document which member is active
  • Use unions in embedded systems for memory efficiency
  • Be careful with pointer arithmetic on unions

Frequently Asked Questions

Q: What is the main difference between structure and union?

A: The main difference is memory allocation. In a structure, each member has its own memory location, while in a union, all members share the same memory location. This means only one member of a union can contain a value at any given time.

Q: How do I determine which member of a union is currently in use?

A: You need to keep track of the active member yourself, typically by using an additional variable (often called a "tag" or "discriminator") that indicates which member is currently being used.

Q: What is the size of a union?

A: The size of a union is equal to the size of its largest member, plus any padding required for alignment. This is because the union must be able to store any of its members.

Q: When should I use unions?

A: Unions are particularly useful in situations where you need to store different types of data in the same memory location, such as in embedded systems for memory optimization, or when implementing variant types or type conversion.
