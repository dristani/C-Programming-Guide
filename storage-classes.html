5. Storage Classes

Understanding storage classes in C programming

Introduction to Storage Classes

Storage classes in C determine the scope, lifetime, and visibility of variables and functions. They define how and where variables are stored in memory and how they can be accessed.

Types of Storage Classes

Storage Class Keyword Scope Lifetime Default Value
Automatic auto Local to block Within block Garbage
Register register Local to block Within block Garbage
Static static Local to block/file Entire program Zero
External extern Global Entire program Zero

Example Program

#include <stdio.h>

// Global variable with extern storage class
extern int global_var;

// Static variable
static int static_global = 0;

void function() {
    // Automatic variable
    auto int auto_var = 10;
    
    // Static variable (retains value between function calls)
    static int static_var = 0;
    static_var++;
    
    // Register variable (suggested to be stored in CPU register)
    register int reg_var = 20;
    
    printf("Auto variable: %d\n", auto_var);
    printf("Static variable: %d\n", static_var);
    printf("Register variable: %d\n", reg_var);
}

int main() {
    // Using different storage classes
    auto int local_var = 5;
    register int counter;
    
    // Initialize global variable
    global_var = 100;
    
    printf("Local variable: %d\n", local_var);
    printf("Global variable: %d\n", global_var);
    printf("Static global: %d\n", static_global);
    
    // Call function multiple times
    function();
    function();
    function();
    
    return 0;
}

Code Explanation

  • extern int global_var; - Declares an external variable
  • static int static_global = 0; - Declares a static global variable
  • auto int auto_var = 10; - Declares an automatic variable (auto is optional)
  • static int static_var = 0; - Declares a static local variable
  • register int reg_var = 20; - Declares a register variable

Storage Class Characteristics

  • auto: Default storage class for local variables
  • register: Suggests compiler to store variable in CPU register
  • static: Preserves value between function calls
  • extern: Used to access global variables across files

Frequently Asked Questions

Q: What is the difference between static and extern storage classes?

A: static variables are limited to their file/block scope, while extern variables can be accessed across multiple files.

Q: When should I use register storage class?

A: Use register for frequently accessed variables, but note that the compiler may ignore the request if registers are not available.

Q: Why use static variables?

A: Static variables maintain their values between function calls and are useful for preserving state or implementing counters.

Q: What happens if I don't specify a storage class?

A: Local variables default to auto, while global variables default to extern.
