23. Command Line Arguments

Understanding command line arguments in C programming

Introduction to Command Line Arguments

Command line arguments allow users to pass parameters to a program when it is executed. In C, these arguments are passed to the main() function through the argc and argv parameters. argc (argument count) is the number of arguments, and argv (argument vector) is an array of strings containing the arguments.

Command Line Argument Parameters

Parameter Description Example
argc Number of command line arguments int argc
argv Array of argument strings char* argv[]
argv[0] Program name "./program"
argv[1] First argument "argument1"

Example Program

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(int argc, char* argv[]) {
    // Basic argument handling
    printf("Program name: %s\n", argv[0]);
    printf("Number of arguments: %d\n", argc - 1);
    
    // Print all arguments
    printf("\nArguments:\n");
    for (int i = 1; i < argc; i++) {
        printf("Argument %d: %s\n", i, argv[i]);
    }
    
    // Argument validation example
    if (argc < 3) {
        printf("\nError: At least 2 arguments required\n");
        printf("Usage: %s <number> <operation>\n", argv[0]);
        return 1;
    }
    
    // Convert string to number
    int number = atoi(argv[1]);
    char* operation = argv[2];
    
    // Process based on operation
    if (strcmp(operation, "square") == 0) {
        printf("\nSquare of %d is %d\n", number, number * number);
    }
    else if (strcmp(operation, "cube") == 0) {
        printf("\nCube of %d is %d\n", number, number * number * number);
    }
    else {
        printf("\nError: Unknown operation '%s'\n", operation);
        printf("Valid operations: square, cube\n");
        return 1;
    }
    
    // Optional arguments example
    int verbose = 0;
    for (int i = 3; i < argc; i++) {
        if (strcmp(argv[i], "-v") == 0) {
            verbose = 1;
        }
    }
    
    if (verbose) {
        printf("\nVerbose mode: Detailed calculation\n");
        printf("Input number: %d\n", number);
        printf("Selected operation: %s\n", operation);
    }
    
    return 0;
}

Code Explanation

  • argc - Count of command line arguments
  • argv - Array of argument strings
  • argv[0] - Always contains program name
  • atoi() - Converts string to integer
  • strcmp() - Compares strings
  • Argument validation ensures correct usage
  • Optional arguments handled with flags

Command Line Argument Characteristics

  • Arguments passed as strings
  • First argument is program name
  • Arguments separated by spaces
  • Quotes can group arguments
  • Case-sensitive by default
  • Can be accessed in any order
  • Can be converted to other types

Command Line Argument Best Practices

  • Validate number of arguments
  • Provide helpful error messages
  • Document argument format
  • Handle conversion errors
  • Use consistent argument order
  • Support both short and long options
  • Consider using argument parsing libraries

Frequently Asked Questions

Q: What is the difference between argc and argv?

A: argc is an integer that represents the number of command line arguments, while argv is an array of strings containing the actual arguments. argc is always at least 1 because argv[0] contains the program name.

Q: How do I handle optional arguments?

A: Optional arguments can be handled using flags (like -v for verbose mode) or by checking the number of arguments. Common approaches include using getopt() function or implementing custom argument parsing logic.

Q: How do I pass arguments containing spaces?

A: To pass arguments containing spaces, enclose them in quotes. For example: ./program "Hello World" will pass "Hello World" as a single argument.

Q: What are some common argument parsing libraries?

A: Common C libraries for argument parsing include getopt (POSIX standard), argp (GNU), and various third-party libraries like argparse. These libraries provide more sophisticated argument handling than basic argc/argv.
